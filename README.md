# Библиотека читшитов JetMetrics

## О проекте
Публичная галерея читшитов по аналитике (сейчас 101 штука, PDF/PNG), которая раньше жила в закрытой Notion-базе. Задача — пересобрать её в кастомную HTML-страницу, встраиваемую в сайт на Tilda, с возможностью управлять контентом без правки кода при каждом обновлении.

## Цели
- Открытая (пока без логина) библиотека читшитов с поиском и фильтрами по тегам
- Обновление силами Клода: новый читшит = новая запись в `data.json`, без визуальной админки
- Встраивание в существующий сайт на Tilda

## Подход
- Полностью статичный сайт: `index.html` + `style.css` + `app.js` + `data.json`, без бэкенда
- Файлы (PDF + PNG-превью) хостятся на GitHub Pages — публичный бесплатный хостинг с обновлением через git push
- Каждый читшит: заголовок, теги (можно несколько), превью (рендер первой страницы PDF), ссылка на PDF, ссылка на исходный пост в Telegram-канале @jetmetrics (где есть)
- Бренд-стиль JetMetrics: зелёный `#0E9C7D`, фиолетовый `#9370DB` как акцент
- В Tilda встраивается через `<iframe>` на GitHub Pages URL (быстрый вариант для демо); нативная вставка без iframe — доработка на потом

## Решения и логика
- **Статика без бэкенда** — потому что Мария хотела «максимально просто, без своего сервера»; обновления идут через файл `data.json`, который готовит Клод
- **GitHub Pages, а не Tilda-загрузка или Google Drive** — обновление = положить файл и запушить, без ручной загрузки через интерфейс; Google Drive даёт капризные прямые ссылки на превью/PDF
- **Свой набор тегов вместо ожидания Notion-экспорта** — Notion был недоступен в сессии, Мария попросила не ждать и предложить свой вариант
- **Серия «How do we use Claude for our job» (7 карточек)** — показана как 7 отдельных карточек, а не одна с мини-каруселью — проще для MVP
- **Заголовки карточек** — оставлен номер статьи (`#044 —`), убран суффикс «RUS»
- **iframe вместо нативного embed для Tilda сегодня** — компромисс ради скорости (демо через час); нативная вставка отложена
